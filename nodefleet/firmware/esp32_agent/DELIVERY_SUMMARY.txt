PROTOCOL SPECIFICATIONS
================================================================================

DEVICE PAIRING:
  Request: POST /api/devices/pair
  Payload: {pairing_code, device_model, firmware_version}
  Response: {device_token, device_id, ws_url}

WEBSOCKET ENDPOINT:
  URL: wss://server.com/device?token=xxx

MESSAGE TYPES:
  - heartbeat (battery, signal, heap, uptime)
  - gps (latitude, longitude, altitude, accuracy)
  - telemetry (custom JSON)
  - command_ack (success status + message)
  - status_report (full device state)

INCOMING COMMANDS:
  - capture_photo
  - capture_video
  - record_audio
  - reboot
  - update_firmware (with URL)
  - get_status
  - set_config (with key/value)

HTTP FALLBACK:
  POST /api/devices/heartbeat (when WebSocket unavailable)

================================================================================
KNOWN LIMITATIONS & NOTES
================================================================================

GPS:
  - First fix may take 30-60 seconds
  - Requires clear sky view
  - Uses SIM7670G built-in GNSS

CAMERA:
  - Requires OV2640 or OV5640 sensor
  - MJPEG video is low-quality (1fps)
  - Maximum resolution VGA (640x480)

AUDIO:
  - Requires I2S microphone hardware
  - Framework included, not fully tested
  - 16kHz sample rate assumed

SD CARD:
  - Must be formatted as FAT32
  - Maximum 32GB recommended
  - Should use Class 10 for best performance

NETWORK:
  - WiFi supports 2.4GHz only (not 5GHz)
  - 4G modem requires valid SIM card
  - Requires roaming enabled for international use

POWER:
  - WiFi mode: ~200mA
  - 4G mode: ~400mA
  - Peak during upload: ~500mA
  - Recommend 3000+ mAh LiPo battery
